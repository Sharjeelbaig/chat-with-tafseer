Pick the right tafseer resource_id before you call the API.

Both POST /chat and GET /tafseer/{resource_id}/chapter/{chapter_number} depend on a Quran.com tafseer resource id. The official source of truth is Quran.com's public tafseer resource list endpoint.

The examples in this docs set default to 169, which Quran.com currently maps to Ibn Kathir (Abridged). The list below reflects the official endpoint fetched on March 29, 2026.

Official Quran.com Endpoint 
https://api.quran.com/api/v4/resources/tafsirs
Most Common Picks

169

Ibn Kathir (Abridged) by Hafiz Ibn Kathir. This is the English tafseer used in the docs examples and the safest default if you want an English explanation flow.

168

Ma'arif al-Qur'an by Mufti Muhammad Shafi. Also English and useful when you want a different English commentary source than the default examples.

817

Tazkirul Quran by Maulana Wahid Uddin Khan. English resource available from the same official Quran.com list.

Language-specific options

Quran.com also exposes Arabic, Bengali, Urdu, Russian, and Kurdish tafseer resources. Use the language column below instead of assuming every id is English.

How This API Uses resource_id
1

Choose a Quran.com tafseer source

Look up the id from Quran.com's official resource list. Your app should store or configure the id it wants instead of hard-coding a guessed value.

2

Send the id to this backend

For verse explanations, include it in the JSON body for POST /chat. For raw chapter tafseer, place it in the route for GET /tafseer/{resource_id}/chapter/{chapter_number}.

3

Do not invent ids

If you need a different tafseer source later, re-check Quran.com's current list endpoint. This backend does not maintain a separate resource catalog.
