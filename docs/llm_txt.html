Agent Guide

Use this API in an LLM workflow without guessing what the backend does.

The backend is small enough that LLMs should follow strict rules: only call chat when the verse is explicit, only reuse threads on that same verse, and fall back to chapter retrieval when raw tafseer content is what you actually need.

When to Call Which Endpoint

Use POST /chat when

  • You know the target verse already.
  • The user wants an explanation, clarification, or follow-up question on that verse.
  • You want the backend to fetch tafseer context and ask the model for a grounded answer.

Use GET /tafseer/{resource_id}/chapter/{chapter_number} when

  • You want raw chapter tafseer content.
  • You are doing indexing, browsing, or source inspection work.
  • You do not need a model answer at all.
Operational Rules
1

Resolve the verse before calling chat

The backend does not infer the verse from free-form user prose. Your agent or client must determine resource_id and verse_key first.

2

Keep thread scope narrow

Reuse a thread_id only for follow-up questions on the same verse. If the verse changes, start a new thread.

3

Treat the answer as grounded model output

The model answer is backed by tafseer context, but it is still model output. The raw source material remains Quran.com content accessed through the backend.

Chat Contract
Field Type Meaning
resource_id integer Tafseer source identifier, for example 169.
verse_key string Verse reference in chapter:verse format, for example 1:1.
message string The user question or follow-up prompt for the same verse.
thread_id string Optional conversation key used for recent-message continuity on that verse.
Recommended Agent Flow
workflow
1. Resolve which verse the user is asking about.
2. Choose the tafseer resource to use.
3. Call POST /chat with resource_id, verse_key, message, and a verse-scoped thread_id.
4. If the user asks a follow-up on the same verse, reuse the same thread_id.
5. If the user switches verses, start a new thread_id.
6. If raw source tafseer is needed, call GET /tafseer/{resource_id}/chapter/{chapter_number} instead.
Error Semantics
Status Meaning
404 The verse or tafseer resource could not be found in the upstream Quran API.
422 The request body is invalid. Common causes are blank text or malformed verse_key.
502 The upstream Quran API failed or the model layer returned a generic generation failure.
503 Ollama is unavailable or the configured model cannot be found.
