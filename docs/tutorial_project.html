Tutorial Project

Build a React tafseer explainer app that feels real, not demo-shaped.

This walkthrough uses Quran.com as the reading source and this backend only for AI explanations. The user never types verse_key or resource_id by hand. Your app derives them from route state and verse interactions.

This guide chooses one stack on purpose: Vite + React + TypeScript + React Router + TanStack Query. Real tutorials get simpler when they stop pretending every choice is interchangeable.

What You’re Building

Route 1: Surah index

Fetch Quran.com chapters and render a navigable list so the user chooses a surah naturally instead of entering raw identifiers.

Route 2: Reader + explainer

Show Arabic ayat and translation on /surah/:chapterNumber. Each verse gets an “Explain” action that calls POST /chat in the background.

Per-verse reply threads

Replies reuse a derived thread_id such as tafseer:169:1:1 so each verse keeps its own short conversation.

Separation of concerns

Quran.com provides reading data. Your backend provides explanation. That split keeps the app honest and easier to reason about.

3. Create Your Data Clients
A

Quran.com client

Use Quran.com for chapter lists, Arabic text, and translation. Translation resource 85 is a working English example for this tutorial.

src/lib/quran.ts
const QURAN_API_BASE = "https://api.quran.com/api/v4";

export async function getChapters() {
  const response = await fetch(`${QURAN_API_BASE}/chapters`);
  if (!response.ok) throw new Error("Failed to load chapters");
  const data = await response.json();
  return data.chapters;
}

export async function getSurahReaderData(chapterNumber: number) {
  const [versesResponse, translationResponse] = await Promise.all([
    fetch(`${QURAN_API_BASE}/quran/verses/uthmani?chapter_number=${chapterNumber}`),
    fetch(`${QURAN_API_BASE}/quran/translations/85?chapter_number=${chapterNumber}`),
  ]);

  if (!versesResponse.ok || !translationResponse.ok) {
    throw new Error("Failed to load surah content");
  }

  const verses = (await versesResponse.json()).verses;
  const translations = (await translationResponse.json()).translations;

  return verses.map((verse: { verse_key: string; text_uthmani: string }, index: number) => ({
    verseKey: verse.verse_key,
    arabic: verse.text_uthmani,
    translation: translations[index]?.text ?? "",
  }));
}
B

Backend client

The frontend never exposes raw verse fields to the user, but it still sends them internally to your backend.

src/lib/tafseer.ts
const TAFSEER_API_BASE =
  import.meta.env.VITE_TAFSEER_API_BASE ?? "https://chat-with-tafseer.vercel.app";

export type ChatRequest = {
  resourceId: number;
  verseKey: string;
  message: string;
  threadId: string;
};

export async function askTafseer(request: ChatRequest) {
  const response = await fetch(`${TAFSEER_API_BASE}/chat`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      resource_id: request.resourceId,
      verse_key: request.verseKey,
      message: request.message,
      thread_id: request.threadId,
    }),
  });

  const data = await response.json();
  if (!response.ok) {
    throw new Error(data.detail ?? "Failed to get tafseer explanation");
  }
  return data as {
    answer: string;
    resource_id: number;
    verse_key: string;
    chapter_number: number | null;
  };
}

5. Build the Surah Index
src/routes/surah-index.tsx
import { Link } from "react-router-dom";
import { useQuery } from "@tanstack/react-query";
import { getChapters } from "../lib/quran";

export function SurahIndexRoute() {
  const chaptersQuery = useQuery({
    queryKey: ["chapters"],
    queryFn: getChapters,
  });

  if (chaptersQuery.isPending) return <p>Loading surahs...</p>;
  if (chaptersQuery.isError) return <p>Unable to load chapters.</p>;

  return (
    <main>
      <h1>Choose a Surah</h1>
      <ul>
        {chaptersQuery.data.map((chapter: any) => (
          <li key={chapter.id}>
            <Link to={`/surah/${chapter.id}`}>
              {chapter.id}. {chapter.name_simple}
            </Link>
          </li>
        ))}
      </ul>
    </main>
  );
}
6. Build the Reader Route

The reader route should fetch verses, render Arabic plus translation, and attach one explanation thread per verse. The user reads first, then asks for help in place.

src/routes/surah-reader.tsx
import { useState } from "react";
import { useParams } from "react-router-dom";
import { useQuery } from "@tanstack/react-query";
import { getSurahReaderData } from "../lib/quran";
import { askTafseer } from "../lib/tafseer";

type ThreadMessage = { role: "user" | "assistant"; content: string };
type ThreadState = Record<string, ThreadMessage[]>;

const TAFSEER_RESOURCE_ID = 169;

function threadIdForVerse(verseKey: string) {
  return `tafseer:${TAFSEER_RESOURCE_ID}:${verseKey}`;
}

export function SurahReaderRoute() {
  const { chapterNumber = "1" } = useParams();
  const [threads, setThreads] = useState<ThreadState>({});
  const [loadingVerse, setLoadingVerse] = useState<string | null>(null);

  const surahQuery = useQuery({
    queryKey: ["surah", chapterNumber],
    queryFn: () => getSurahReaderData(Number(chapterNumber)),
  });

  async function submitForVerse(verseKey: string, message: string) {
    const threadId = threadIdForVerse(verseKey);
    setLoadingVerse(verseKey);
    setThreads((current) => ({
      ...current,
      [verseKey]: [...(current[verseKey] ?? []), { role: "user", content: message }],
    }));

    try {
      const result = await askTafseer({
        resourceId: TAFSEER_RESOURCE_ID,
        verseKey,
        message,
        threadId,
      });

      setThreads((current) => ({
        ...current,
        [verseKey]: [...(current[verseKey] ?? []), { role: "assistant", content: result.answer }],
      }));
    } finally {
      setLoadingVerse(null);
    }
  }

  if (surahQuery.isPending) return <p>Loading surah...</p>;
  if (surahQuery.isError) return <p>Unable to load this surah.</p>;

  return (
    <main>
      {surahQuery.data.map((verse) => (
        <section key={verse.verseKey}>
          <p>{verse.arabic}</p>
          <p dangerouslySetInnerHTML={{ __html: verse.translation }} />
          <button onClick={() => submitForVerse(verse.verseKey, "Explain this verse briefly.")}>
            {loadingVerse === verse.verseKey ? "Explaining..." : "Explain"}
          </button>
          <ExplanationThread
            verseKey={verse.verseKey}
            messages={threads[verse.verseKey] ?? []}
            onReply={submitForVerse}
          />
        </section>
      ))}
    </main>
  );
}
7. Add Replies to Each AI Explanation

Why replies belong to the verse card

Each verse is its own context boundary. Putting the explanation thread directly under the verse keeps the user anchored to the ayah they are asking about.

  • Use the same thread_id for follow-up replies on the same verse.
  • Store the visible transcript in local component state or query cache.
  • Never reuse the same thread between different verses.

Reply component sketch

explanation-thread.tsx
export function ExplanationThread({
  verseKey,
  messages,
  onReply,
}: {
  verseKey: string;
  messages: { role: "user" | "assistant"; content: string }[];
  onReply: (verseKey: string, message: string) => Promise<void>;
}) {
  return (
    <div>
      {messages.map((message, index) => (
        <p key={`${verseKey}-${index}`}>
          <strong>{message.role === "assistant" ? "AI" : "You"}:</strong> {message.content}
        </p>
      ))}
      <button onClick={() => onReply(verseKey, "How does this connect to the next verse?")}>
        Reply
      </button>
    </div>
  );
}
8. Handle Loading, Errors, and Restraint

Do this

  • Disable the explain button while a verse request is in flight.
  • Show a small error under the verse if the backend returns 404, 502, or 503.
  • Keep translation and tafseer clearly separated in the UI.
  • Prefill the first explain prompt with user-friendly language, not raw verse metadata.

Avoid this

  • Exposing a form where the user must type resource_id or verse_key.
  • Using one global thread for the entire surah reader.
  • Letting AI output replace the actual Quran text or translation in the reader.
  • Pretending the explanation is source text instead of model output grounded in tafseer.

Build Outline

  • Use Quran.com for reading data.
  • Use this backend only for explanation calls.
  • Key each thread by verse identity.
  • Keep AI output as a companion layer, not the reader itself.

Ship Checklist

  • Reader route loads Arabic plus translation.
