Input Guide

A verse_key is just chapter:verse.

Quran.com's official verse-by-key docs define verse_key from the chapter number and ayah number. In practical terms, 1:1 means chapter 1, verse 1, and 2:255 means chapter 2, verse 255.

This backend expects explicit verse input. It does not guess a verse from prose like "tell me about Ayat al-Kursi". Your client should resolve the verse first, then send the exact verse_key.

How to Read It

Before the colon

The number on the left is the chapter number. Example: in 18:10, the chapter is 18.

After the colon

The number on the right is the verse number inside that chapter. Example: in 18:10, the verse is 10.

Quick Examples

Where to Get verse_key Values
1

Use Quran.com verse payloads

Quran.com verse APIs include verse_key on each verse object. In the tutorial project, the reader route gets it directly from Quran.com's verse data instead of making the user type it.

2

Keep it unchanged

If Quran.com returns 18:10, send 18:10 to POST /chat. Do not convert it to names, slugs, or other formats before sending it.

3

Derive chapter_number when needed

This backend echoes chapter_number in chat responses, but the input still remains verse_key. The chapter is simply the number before the colon.

Common Mistakes
Input Problem
1-1Wrong separator. Use a colon, not a dash.
Al-Fatiha:1Chapter names are not accepted here. Use numeric chapter ids.
1:0Verse numbering starts at 1 inside a chapter.
message onlyThis API does not infer the verse. You must send verse_key explicitly.

Rule of Thumb

  • chapter:verse
  • Left side = chapter.
  • Right side = verse.
  • 1:1 means first chapter, first verse.
