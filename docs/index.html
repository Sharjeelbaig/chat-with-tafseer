Developer Toolkit

Build verse-grounded tafseer experiences with a small, honest backend.

Chat with Tafseer is a FastAPI service for two jobs: verse-scoped AI explanations through POST /chat and chapter-level tafseer retrieval through GET /tafseer/{resource_id}/chapter/{chapter_number}. It pulls classical commentary from Quran.com, shapes the context, and asks Ollama to answer only from that material.

What this project is for

Use this backend when you want a client app to ask focused questions about a specific verse without making the user manually paste tafseer text or manage Quran data plumbing.

The current design is intentionally narrow. The API does not parse free-form surah requests, does not implement authentication, and does not pretend to be a general Islamic chatbot. It expects explicit verse context and stays there.

How clients should think about it

01

Resolve the verse

Your client chooses the tafseer resource and computes the verse_key.

02

Call the backend

Send resource_id, verse_key, a user question, and an optional thread_id to /chat.

03

Render the answer

Display the response with the source verse and keep the same thread_id for follow-up replies on that verse.

Ecosystem

Quran.com

The backend uses Quran.com’s public API for tafseer by ayah and by chapter. The tutorial page also uses Quran.com directly for chapter lists, Arabic text, and translation data in the frontend reader experience.

Ollama

Ollama is the answer generator, not the source of truth. The service fetches tafseer first, strips HTML, truncates to a safe context window, and instructs the model to answer from that supplied commentary only.

How to Run Locally
01

Clone the repository

Pull down the codebase from GitHub to your local machine.

git clone https://github.com/Sharjeelbaig/chat-with-tafseer
02

Sync dependencies

Install the required Python packages and setup the virtual environment.

uv sync
03

Configure your environment

Copy the example environment file and set your API keys.

cp .env.example .env

# Edit .env and set your keys:
# OLLAMA_API_KEY="Super_Secret"
04

Run the backend

Start the development server using uv.

uv run uvicorn main:app --reload
What This Is and Is Not

This is

  • A backend API you can call from your own web or mobile app.
  • A developer toolkit for verse-scoped tafseer explanations backed by Quran.com and Ollama.
  • A clean starting point for building reader apps, study tools, or guided tafseer experiences.

This is not

  • A published JavaScript SDK with client packages and generated bindings.
  • An authentication platform, multi-tenant API gateway, or user account system.
  • A broad free-form question parser that guesses surahs or verses from arbitrary prose.
