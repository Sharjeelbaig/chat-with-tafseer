Tafseer Retrieval

Fetch chapter tafseer directly when you do not need a model answer.

GET /tafseer/{resource_id}/chapter/{chapter_number} proxies chapter-level tafseer from Quran.com. It does not chat, does not maintain thread state, and does not call Ollama.

GET /tafseer/{resource_id}/chapter/{chapter_number}

Choose this endpoint when you want source tafseer material by chapter. If you want a user-facing explanation for a single verse, use POST /chat instead.

Path Parameters
Parameter Type Description
resource_id integer The tafseer source to query, for example 169.
chapter_number integer The surah number to fetch, from 1 through 114.
Sample Response Shape
200 OK
{
  "tafsirs": [
    {
      "verse_key": "1:1",
      "text": "<p>Chapter tafseer content...</p>"
    }
  ]
}
When to Use This Endpoint

Good fit

  • Indexing or storing source tafseer by chapter.
  • Manual reader or editor workflows.
  • Bulk chapter inspection without model generation.

Wrong fit

  • Verse-level Q&A with follow-up replies.
  • Any feature that needs thread continuity.
  • A frontend flow where the user expects a natural-language answer.
Error Handling
Status When it happens
404 The tafseer resource is missing or the chapter number is invalid upstream.
502 Quran.com failed to respond successfully or the transport request failed.
