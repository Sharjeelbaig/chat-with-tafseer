Chat Endpoint

Send a verse-grounded tafseer message.

POST /chat answers a user question using only the tafseer for the verse you specify. It is designed for explicit verse input, not free-form surah guessing.

POST /chat

No authentication is implemented in this repository. Send JSON directly. The backend will fetch tafseer from Quran.com using the supplied resource_id and verse_key, then send the cleaned context to Ollama.

Request Body
Parameter Type Description Required
resource_id integer The tafseer source to use. Example: 169 for Ibn Kathir (Abridged). required
verse_key string Verse identity in chapter:verse format. Example: 1:1. required
message string The user’s question or follow-up message about that verse. required
thread_id string Optional session key for follow-up replies. Reuse it only for the same verse if you want context continuity. optional
Successful Response
200 OK
{
  "answer": "According to the supplied tafseer context, the verse opens by praising Allah...",
  "resource_id": 169,
  "verse_key": "1:1",
  "chapter_number": 1
}
Response Fields
Field Type Description
answer string The model answer grounded in the fetched tafseer text.
resource_id integer Echoes the tafseer resource used for the response.
verse_key string Echoes the resolved verse key used for context.
chapter_number integer Derived from the verse key so clients can keep reader state aligned.
Behavior Notes

Explicit verse input only

The backend does not infer the verse from free-form user prose. Resolve the verse in your client and send resource_id plus verse_key explicitly.

Thread reuse rules

Using the same thread_id on the same verse keeps recent chat history. Reusing that thread on a different verse resets the stored context to the new verse.

Error Handling
Status When it happens
404 The Quran.com tafseer resource or verse does not exist for the request.
422 The body is invalid, for example a blank message or malformed verse_key.
502 The upstream Quran API failed or the model returned an unexpected internal error.
503 Ollama is unavailable or the configured model cannot be reached.
